# POLITIQUE RELATIVE AUX COOKIES ET AU STOCKAGE LOCAL

**Dernière mise à jour : [DATE DU JOUR]**

## 1. Qu'est-ce qu'un "Cookie" ?
Dans le contexte d'une application mobile, le terme "cookie" couvre les technologies permettant de stocker des informations sur votre appareil (stockage local, cache, identifiants temporaires).

## 2. Comment utilisons-nous ces technologies ?
L'application **LovMap** utilise le stockage local de votre appareil pour les finalités suivantes :

### 2.1. Fonctionnement Technique (Strictement Nécessaire)
Ces éléments sont indispensables pour que l'Application fonctionne correctement. Ils ne requièrent pas votre consentement.
- **Session Utilisateur (Authentification) :** Nous stockons un "token" (clé numérique) sécurisé lorsque vous vous connectez. Cela vous évite de devoir entrer votre email et mot de passe à chaque ouverture de l'Application. Ce token est géré par Supabase.
- **Préférences :** Nous pouvons stocker vos choix de configuration (ex: thème sombre/clair, langue, activation des sons).

### 2.2. Cartographie
L'Application utilise des services de cartographie (Google Maps ou Apple Maps selon votre appareil). Ces services peuvent stocker des fichiers en cache pour améliorer le chargement des cartes.

## 3. Gestion de vos préférences
Puisque nous n'utilisons que des traceurs strictement nécessaires au fonctionnement du service (authentification), il n'y a pas d'option pour les désactiver dans l'Application sans empêcher son fonctionnement.

Vous pouvez cependant effacer les données de l'Application ou désinstaller l'Application via les paramètres de votre système d'exploitation (iOS ou Android), ce qui supprimera toutes les données stockées localement.
